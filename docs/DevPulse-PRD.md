## DevPulse 产品需求文档（PRD）

**产品名称：** DevPulse — 开发者内容社区平台  
**版本：** v3.0  
**最后更新：** 2026-06-14  
**定位：** 面向开发者的技术内容社区（类掘金/Dev.to），用户可以发布技术文章、参与讨论、关注作者、收藏内容、接收通知。

---

### 一、目标用户

平台面向三类用户群体。技术爱好者（读者）希望在社区中浏览优质技术文章、点赞收藏、评论交流、关注喜欢的作者。技术创作者（作者）希望拥有一个富文本编辑器来发布技术文章，管理自己的内容，积累粉丝和影响力。平台运营方（管理员）需要审核管理内容、维护用户秩序、查看平台运营数据。

---

### 三、站点地图与导航结构

```
/                        首页（文章列表）
/tags                    标签广场
/article/:slug           文章详情
/editor                  写文章（新建）
/editor/:id              编辑文章
/search                  搜索
/login                   登录
/register                注册
/users/:id               用户公开主页
/settings                个人设置
/bookmarks               我的收藏
/notifications           通知中心
/admin                   管理后台-仪表盘
/admin/users             管理后台-用户管理
/admin/articles          管理后台-文章管理
/admin/tags              管理后台-标签管理
```

**公共导航栏（Header）** 包含：Logo（链接到首页）、首页导航、标签导航、全局搜索框（桌面端）、写文章按钮（需登录，AUTHOR+）、通知铃铛（需登录，显示未读红点）、用户头像下拉菜单（个人主页、我的收藏、设置、管理后台(仅ADMIN)、退出登录）。未登录时显示"登录"和"注册"按钮。移动端有汉堡菜单。

**管理后台导航** 为独立侧边栏布局，包含四个导航项：仪表盘、用户管理、文章管理、标签管理。管理后台不使用公共 Header，有自己的页面标题。

#### 4.1 首页 — 文章列表页

**URL:** `/`  
**访问权限:** 所有人（含游客）  
**布局:** 左右两栏，主内容区 + 右侧边栏（桌面端显示，移动端隐藏侧边栏）

**主内容区：**

顶部有三个排序标签页：「最新」（按发布时间倒序）、「热门」（按阅读量倒序）、「最多点赞」（按点赞数倒序）。选中的标签页以蓝色高亮。

下方为文章卡片列表，每张卡片显示：作者头像 + 昵称 + 发布时间（相对时间，如"3天前"）、文章标题（可点击跳转详情）、文章摘要（截断至150字符，无摘要则不显示）、标签徽章行（可点击按标签筛选）、底部统计行（阅读量 + 点赞数 + 评论数，带图标）。如果文章有封面图，卡片右侧显示缩略图（移动端隐藏）。鼠标悬停卡片时有轻微阴影浮起效果。

列表底部有分页器（接入后端 API 后显示），支持翻页。

**右侧边栏：**

「热门标签」模块，展示所有标签的徽章，点击标签后首页文章列表按该标签筛选（URL 变为 `/?tag=react`），选中的标签以蓝色环框高亮。第一个选项为「全部」，点击清除筛选。

「关于 DevPulse」简介模块。

**空状态：** 如果筛选后无文章，显示"暂无文章"提示。

**业务规则：** 只展示 status=PUBLISHED 且未软删除的文章。

#### 4.2 文章详情页

**URL:** `/article/:slug`  
**访问权限:** 所有人  
**布局:** 单栏居中，最大宽度限制

**页面结构（从上到下）：**

**文章头部：** 大标题、作者信息行（头像 + 昵称(可点击跳转主页) + 发布日期 + 阅读时长 + 阅读量）、标签徽章行。

**文章正文：** HTML 富文本渲染区域，使用 prose 排版样式（支持 h1-h3、段落、代码块、列表、引用、图片、表格、链接等）。代码块为深色背景 + 等宽字体。

**操作栏（固定在正文下方）：** 点赞按钮（心形图标 + 数字，已点赞时变红）、收藏按钮（书签图标 + 文字，已收藏时变蓝"已收藏"）、分享按钮。三个按钮水平排列。未登录时点赞/收藏按钮禁用（灰色，tooltip 提示"请先登录"）。

**作者操作：** 如果当前登录用户是文章作者，头部右侧显示「编辑」按钮，跳转到 `/editor/:id`。

**评论区（正文下方）：** 已登录用户显示评论输入框 + "发表评论"按钮。未登录用户显示"登录后参与评论"提示（链接到登录页）。评论列表按时间倒序展示，每条评论显示：评论者头像 + 昵称 + 时间 + 内容 + 点赞数 + 回复按钮。支持两级嵌套（顶级评论 + 回复），回复缩进显示在父评论下方。每条评论旁有点赞按钮（toggle）和删除按钮（仅评论作者、文章作者、ADMIN 可见）。

**空状态：** 评论区无评论时显示"暂无评论，来说两句吧"。

**业务规则：** 每次访问详情页，阅读量 +1（后端用 Redis 缓冲，不直接写 DB）。

#### 4.3 文章编辑器页

**URL:** `/editor`（新建）、`/editor/:id`（编辑）  
**访问权限:** 需登录，AUTHOR 及以上  
**布局:** 全屏宽度，无侧边栏

**页面结构：**

顶部有标题输入框（大号字体，placeholder "文章标题"）。

标题下方有摘要输入框（单行，placeholder "文章摘要（可选，用于列表展示）"）。

编辑工具栏：加粗、斜体、二级标题、三级标题、无序列表、有序列表、代码块、撤销、重做。每个按钮是图标按钮，当前格式激活时按钮变蓝色。

正文区域为 Tiptap 富文本编辑器，placeholder "开始写作..."。

底部操作区：「保存草稿」按钮（次要样式）、「发布」按钮（主要样式）。编辑已有文章时，"发布"按钮文字变为"更新"。

**权限守卫：** 未登录 → 自动跳转 `/login`。READER 角色 → 显示"需要 AUTHOR 权限"提示。

**业务规则：** 新建文章时自动生成 slug（标题转小写 + 空格转连字符 + 随机后缀防冲突）。阅读时长按纯文本字数 / 200 字每分钟计算。草稿状态的文章不出现在公共列表中。

#### 4.5 登录页

**URL:** `/login`  
**访问权限:** 未登录用户  
**布局:** 居中卡片（AuthLayout），无 Header/Footer

**表单字段：** 邮箱（必填，校验邮箱格式）、密码（必填，不能为空）。

**交互：** 点击「登录」按钮提交，按钮显示 loading 旋转状态。登录成功后自动跳转首页。登录失败在表单顶部显示红色错误横幅（如"邮箱或密码错误"）。表单底部有"还没有账号？立即注册"链接。

**业务规则：** 被封禁的用户登录时返回 403，显示"您的账号已被封禁"。不暴露具体是邮箱不存在还是密码错误（统一提示"邮箱或密码错误"）。

**URL:** `/register`  
**访问权限:** 未登录用户  
**布局:** 居中卡片（AuthLayout）

#### 4.10 通知中心页

**URL:** `/notifications`  
**访问权限:** 需登录  
**布局:** 单栏居中

标题"通知"带铃铛图标，旁边红色未读数量徽章（如 `3`）。

标题右侧有「全部已读」按钮（幽灵样式），仅在有未读通知时显示。

**通知列表：** 每条通知显示：触发者头像（可点击跳转其主页）、通知类型图标（用 emoji 区分）、通知内容文本（如"Cool Dev 赞了你的文章《深入理解 React Hooks》"）、发布时间（相对时间）。

**已读/未读状态：** 未读通知有浅蓝色背景 + 左侧蓝色圆点指示器。点击任意通知将其标记为已读。

**6种通知类型及图标：**
- ARTICLE_LIKED: ❤ — 有人赞了你的文章
- COMMENT_RECEIVED: 💬 — 有人评论了你的文章
- COMMENT_REPLIED: ↩ — 有人回复了你的评论
- COMMENT_LIKED: ❤ — 有人赞了你的评论
- USER_FOLLOWED: 👤 — 有人关注了你
- ARTICLE_PUBLISHED: 📝 — 你关注的作者发布了新文章

**空状态：** "暂无通知"。

**业务规则：** 不给自己发通知（比如自己给自己文章点赞不产生通知）。

#### 4.11 搜索页

**URL:** `/search`  
**访问权限:** 所有人  
**布局:** 单栏居中

顶部为大型搜索输入框（自动聚焦），带搜索图标。

**搜索结果列表：** 每条结果显示：文章标题（搜索命中的关键词用 `<mark>` 高亮）、文章摘要、作者昵称、发布日期。点击结果跳转到文章详情页。

**业务规则：** 使用 PostgreSQL 全文搜索（tsvector + tsquery），标题权重最高。只搜索 PUBLISHED 状态的文章。

### 五、核心用户流程

#### 5.1 注册 → 发表第一篇文章

```
注册页(/register)
  │ 填写邮箱/用户名/密码 → 注册成功
  ▼
首页(/) — 已登录状态
  │ 导航栏出现"写文章"按钮
  ▼
编辑器页(/editor)
  │ 输入标题、撰写正文、选择标签
  │ 点击"保存草稿" → 文章状态 DRAFT，保存在数据库
  │ 或点击"发布" → 文章状态 PUBLISHED，出现在公共列表
  ▼
文章详情页(/article/slug) — 可查看自己发布的文章
```

#### 5.2 浏览 → 互动流程

```
首页 → 点击文章卡片
  ▼
文章详情页
  │ 点击 ❤ 点赞按钮 → toggle（红/灰切换），文章 likeCount ±1
  │ 点击 🔖 收藏按钮 → toggle（蓝/灰切换）
  │ 在评论框输入内容 → 点击"发表评论" → 评论列表顶部出现新评论
  │ 点击评论的"回复" → 展开回复输入框 → 提交后作为子评论显示
  ▼
通知产生（异步，通过 BullMQ 队列）
  │ 作者收到 COMMENT_RECEIVED 通知
  │ 作者收到 ARTICLE_LIKED 通知
```

#### 5.3 关注流程

```
用户主页(/users/:id)
  │ 点击"关注"按钮
  ▼
按钮变为"已关注"，对方收到 USER_FOLLOWED 通知
  │ 再次点击 → 取消关注，按钮恢复"关注"
```

#### 5.4 管理后台流程

```
管理员登录 → 头像下拉菜单出现"管理后台"入口
  ▼
/admin 仪表盘 — 查看统计数据
  ▼
/admin/users — 搜索用户 → 修改角色 → 封禁/解封
  ▼
/admin/articles — 查看/删除违规文章
  ▼
/admin/tags — 创建新标签 / 删除废弃标签
```

### 六、全局交互规范

**响应式断点：** 移动端（<768px）隐藏右侧边栏，导航栏变为汉堡菜单，表格变为可横向滚动。桌面端（>=768px）完整展示。

**加载状态：** 所有按钮点击后显示 loading 旋转图标，禁用重复提交。页面数据加载中显示骨架屏或 loading 指示器。

**错误处理：** 表单校验错误在对应字段下方显示红色文字。API 错误在表单顶部显示红色横幅。网络错误显示"网络连接失败，请重试"。

**认证过期：** accessToken 过期后自动尝试用 refreshToken 刷新。刷新也失败时清除登录状态并跳转到登录页。并发请求的刷新会自动排队，避免多次重复刷新。

**相对时间：** 发布时间统一显示为相对时间格式："刚刚"、"N分钟前"、"N小时前"、"N天前"、超过30天显示具体日期。

**数字格式化：** 超过 1000 显示为 "1.2k"，超过 10000 显示为 "1.2w"。

### 七、通知系统规则

通知通过 BullMQ 异步创建，不阻塞主业务流程。

| 触发操作 | 通知类型 | 接收人 | 内容模板 |
|---|---|---|---|
| 点赞文章 | ARTICLE_LIKED | 文章作者 | "{actor} 赞了你的文章《{articleTitle}》" |
| 评论文章 | COMMENT_RECEIVED | 文章作者 | "{actor} 评论了你的文章：{commentSnippet}" |
| 回复评论 | COMMENT_REPLIED | 父评论作者 | "{actor} 回复了你的评论：{replySnippet}" |
| 点赞评论 | COMMENT_LIKED | 评论作者 | "{actor} 赞了你的评论" |
| 关注用户 | USER_FOLLOWED | 被关注者 | "{actor} 开始关注你" |
| 发布文章 | ARTICLE_PUBLISHED | 作者的粉丝 | "{actor} 发布了新文章《{articleTitle}》" |

**防重复：** 同一用户对同一文章多次点赞只发一次通知（toggle 取消后重新点赞也不重复发）。短时间内（5分钟）同一 actor 对同一 recipient 的同类通知合并为一条。

**自操作过滤：** actor === recipient 时不创建通知。

### 八、数据模型概览

```
User (用户)
├── id, email(唯一), username(唯一), passwordHash
├── displayName, avatar, bio, role(READER/AUTHOR/ADMIN)
├── isBanned, bannedAt, banReason
└── createdAt, updatedAt

Article (文章)
├── id, title, slug(唯一), content, summary, coverImage
├── status(DRAFT/PUBLISHED/ARCHIVED), viewCount, likeCount, commentCount
├── readTimeMinutes, version(乐观锁), searchVector(全文搜索)
├── author → User, tags → Tag[](多对多)
├── publishedAt, deletedAt(软删除), createdAt, updatedAt
└── @@index(status+publishedAt), @@index(authorId), @@index(slug)

Tag (标签)
├── id, name(唯一), slug(唯一), description, color, articleCount
└── articles → Article[](多对多)

Comment (评论)
├── id, content, article → Article, author → User
├── parentId(自关联，顶级为null), replies → Comment[]
└── createdAt, updatedAt

Like (文章点赞)
├── id, user → User, article → Article
├── createdAt
└── @@unique(userId, articleId)

CommentLike (评论点赞)
├── id, user → User, comment → Comment
└── @@unique(userId, commentId)

Bookmark (收藏)
├── id, user → User, article → Article
└── @@unique(userId, articleId)

Follow (关注)
├── id, follower → User, following → User
└── @@unique(followerId, followingId)

Notification (通知)
├── id, type(6种枚举), recipient → User, actor → User
├── articleId, commentId, content, isRead
└── @@index(recipientId+isRead+createdAt)
```

### 九、非功能性需求

**性能：** 文章列表接口响应时间 < 200ms。全文搜索响应时间 < 500ms。首页加载（含前端资源）< 2s。

**安全：** 密码 bcrypt 哈希存储（cost=12），不可逆。JWT 双令牌，accessToken 15 分钟过期，refreshToken 7 天过期并 Token Rotation。所有输入通过 class-validator 校验。API 限流（默认 60 次/分钟，认证接口 10 次/分钟）。XSS 防护（富文本内容需 sanitize）。

**可用性：** 表单校验实时反馈，错误信息明确。操作成功/失败有明确视觉提示。空状态有引导文案。所有可点击元素有 hover 状态。
