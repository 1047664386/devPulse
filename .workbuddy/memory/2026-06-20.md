## 修复忘记密码邮件发送功能

### 问题
1. **邮件发送失败** `connect ECONNREFUSED ::1:1025`
2. **失败后有 60s 冷却 + 前端无失败提示**（用户傻等）

### 根因
- **Mailpit 容器未启动**（邮件发不出去的直接原因）
- **`SMTP_HOST=localhost` 在 macOS 下解析为 IPv6 `::1`**，连接 Mailpit 时报 ECONNREFUSED
- **`mail.service.ts` 开发模式 catch 块静默吞掉 SMTP 错误**（打印 warn 后 return 视为成功），导致 `auth.service.ts` 误以为发送成功 → 错误设置冷却期 + 前端误显示"邮件已发送"。这破坏了代码里本已写对的"失败不设冷却"逻辑。

### 修复
1. 启动 Mailpit 容器：`docker compose up -d mailpit`（端口 1025 SMTP / 8025 Web UI）
2. `apps/api/.env` + `.env.example` + 根目录 `.env.example`：`SMTP_HOST=localhost` → `127.0.0.1`（强制 IPv4）
3. 重写 `apps/api/src/common/mail/mail.service.ts`：
   - 开发模式默认 host 改为 `127.0.0.1`
   - **SMTP 失败时抛错**（不再静默吞掉），保留控制台打印重置链接作为调试辅助
   - 这样 auth.service 的 catch 会触发：清理令牌 + 不设冷却 + 返回 ErrMailSendFailed(20028)
4. `auth.service.ts` 无需改动（失败处理逻辑本就正确）

### 验证结果
- ✅ 成功场景：调用忘记密码接口 → Mailpit 真实收到邮件（author@devpulse.com）
- ✅ 失败场景：停 Mailpit → 接口返回 `{"code":20028,"message":"邮件发送失败，请稍后重试或联系管理员"}` → 立即重试仍返回 20028（未设冷却）
- ✅ 前端拦截器（api.ts 第 30-37 行）能识别 HTTP 200 + code!=0 转为 reject，getApiError 提取 message 显示

### 关键经验
- macOS + Node.js 18+ 下 `localhost` 优先解析 IPv6 `::1`，本地服务连接要用 `127.0.0.1`
- 开发模式的"静默兜底"会掩盖真实失败，破坏上层错误处理逻辑——兜底应保留调试信息但不应假装成功

### 环境变量「本地 vs 线上」配置文档
用户疑问：`.env` 被 gitignore 忽略，线上怎么读配置？补充文档到 `docs/DevPulse-开发手册.md` Step 3 环境变量章节：
- `.env` 只是本地便利文件，NestJS 实际读 `process.env`
- 线上三种注入方式：Docker environment/env_file、PM2 ecosystem、云平台环境变量
- 本项目 `docker-compose.yml` 第 12 行 `${POSTGRES_PASSWORD}` 即生产模式设计（无默认值，必须注入）
- `env.validation.ts` 启动时校验必填变量，缺失直接阻断启动
- 最佳实践：密钥不进 git、敏感变量走密钥管理、环境隔离
